Core Technologies

  • Backend: FastAPI (async Python web framework)
  • Frontend: React 18 + Vite (modern build tool)
  • LLM: Claude (Anthropic API for agent reasoning)
  • Real-Time: WebSocket for streaming agent execution events
  • Database: PostgreSQL for run history & persistence

🤖 The Five AI Agents

1. Strategist Agent

Purpose

Generates comprehensive marketing campaign briefs with channel mix, budget allocation, KPIs, and strategic rationale.

Input Parameters

Parameter Type Description Example
campaign_goal string The campaign objective "Generate 50 MQLs for Lumen Pro from mid-market CFOs"
budget_usd float Total budget in USD 50000
timeline_days integer Campaign duration 90
target_segment string Target audience description "VP of Operations at Fortune 500 companies"

Execution Process

Step 1: Parse campaign request and extract key objectives
Step 2: Search knowledge base for brand guidelines and messaging
Step 3: Query database for past campaign performance on similar segments
Step 4: Retrieve channel benchmarks from knowledge base
Step 5: Generate multi-channel strategy with quantified recommendations
Step 6: Output structured JSON with channel mix, budget split, KPIs, and rationale

Output Schema

{ "campaign_goal": "Generate 50 MQLs...", "recommended_channels": ["LinkedIn", "Email", "Content"], "budget_allocation": { "LinkedIn": 25000, "Email": 15000, "Content": 10000 }, "primary_kpis": ["MQL Volume", "CAC", "ROAS"], "timeline_recommendation": "Q4 2026", "key_risks": ["Audience saturation", "Competitive activity"], "rationale": "LinkedIn drives highest engagement for this persona..." }

2. Creative Agent

Purpose

Generates multiple ad copy variants optimized for specific channels, personas, and messaging angles. Includes brand safety validation.

Input Parameters

Parameter Type Description
channel string facebook, instagram, reels, linkedin, email, etc.
product string Product or service being advertised
persona string Target persona name or description
tone string friendly, professional, urgent, educational
key_message string Core value proposition or message
num_variants integer Number of creative variants (1-5)

Safety Features

  • Validates copy against brand guidelines
  • Checks for prohibited phrases and messaging violations
  • Ensures brand voice consistency
  • Flags potentially problematic content

3. Analyst Agent

Purpose

Answers natural-language marketing analytics questions by generating SQL queries, executing them, and providing business insights with proper context.

Input

question: Natural language question like "What is the ROAS by channel for Q1 2026?"

Allowed Tables

  • campaigns - Master campaign data
  • campaign_briefs - Strategic briefs from Strategist
  • creatives - Ad variants from Creative Agent
  • performance_events - Daily metrics (impressions, clicks, conversions, spend)
  • optimization_recommendations - Recommendations from Optimizer
  • eval_runs - Model evaluation results

Safety Measures

  • Only SELECT queries allowed
  • Query validation before execution
  • Result anomaly detection (ROAS > 10x flagged)
  • Data quality assessment included

4. Optimizer Agent

Purpose

Analyzes campaign performance and generates quantified optimization recommendations based on benchmarks, historical data, and statistical significance.

Input Parameters

Parameter Type Description
campaign_id integer Campaign to optimize
campaign_name string Human-readable name
remaining_budget_usd float Unspent budget
days_remaining integer Campaign flight time left

Analysis Approach

1. Retrieves current performance metrics by channel
2. Compares against industry benchmarks
3. Identifies underperforming channels (statistically significant)
4. Validates findings against historical patterns
5. Generates budget reallocation recommendations with expected ROI impact

5. A/B Testing Agent

Purpose

Designs statistically valid experiments using stratified random sampling, power analysis, and covariate balance validation.

Key Features

  • Power Analysis: Calculates required sample sizes for desired statistical power
  • Stratification: Divides population by industry, size, DMA tier, advertising experience
  • Balance Validation: Ensures covariate balance using chi-square and Welch's t-test
  • Group Assignment: Proportional stratified random sampling
  • Diagnostics: Comprehensive balance reports and standardized effect sizes

Inputs

  • experiment_name - Human-readable identifier
  • baseline_conversion_rate - Expected baseline (0-1)
  • minimum_detectable_effect - Minimum lift to detect (relative or absolute)
  • desired_power - Statistical power goal (0.5-0.99, typically 0.8)
  • significance_level - Type I error rate (typically 0.05)
  • test_fraction - Proportion to assign to treatment (0.1-0.9)

⚡ WebSocket Real-Time Event Streaming

How Event Streaming Works

When an agent starts executing, it emits events throughout its lifecycle. These are broadcast to connected WebSocket clients in real-time, providing live visibility into what the agent is doing.

Connection Protocol

// Frontend connects after receiving run_id const wsProtocol = location.protocol === 'https:' ? 'wss' : 'ws'; const wsUrl = `${wsProtocol}://${API_HOST}/ws/agent-events/${run_id}`; const ws = new WebSocket(wsUrl); ws.onmessage = (event) => { const agentEvent = JSON.parse(event.data); console.log('Event:', agentEvent); };

Event Types & Examples

1. Tool Call Event

{ "type": "tool_call", "tool_name": "search_knowledge_base", "input": { "query": "brand voice guidelines", "n_results": 5 }, "timestamp": 1713811593.5 }

2. Tool Result Event

{ "type": "tool_result", "tool_name": "search_knowledge_base", "output": [ { "doc_id": "brand_001", "content": "Brand voice is professional yet approachable..." } ], "timestamp": 1713811594.2 }

3. Reasoning Event

{ "type": "reasoning", "thinking": "The user wants a LinkedIn strategy for VP of Operations...", "timestamp": 1713811595.0 }

4. Completion Event

{ "type": "completed", "success": true, "total_turns": 5, "timestamp": 1713811620.3 }

5. Error Event

{ "type": "error", "message": "Failed to query database: connection timeout", "timestamp": 1713811605.1 }

Event Flow During Execution

Agent Execution Timeline: T+0.0s: reasoning event ├─ Analyzing campaign goal and constraints │ T+0.5s: tool_call event (search_knowledge_base) │ T+1.2s: tool_result event │ T+1.5s: tool_call event (get_campaign_history) │ T+2.8s: tool_result event │ T+3.0s: tool_call event (get_benchmark_data) │ T+4.1s: tool_result event │ T+4.5s: reasoning event ├─ Synthesizing findings into recommendations │ T+5.2s: completed event └─ Agent finished, run_id stored in database

💾 Database Persistence & Run History

Agent Runs Table Schema

CREATE TABLE agent_runs ( id SERIAL PRIMARY KEY, run_id UUID NOT NULL UNIQUE, agent_name VARCHAR(50), model VARCHAR(100), max_turns INTEGER, input_params JSONB, success BOOLEAN, output JSONB, error TEXT, tool_calls_made JSONB, total_input_tokens INTEGER, total_output_tokens INTEGER, latency_ms FLOAT, trace_url TEXT, created_at TIMESTAMP DEFAULT NOW(), updated_at TIMESTAMP DEFAULT NOW() );

Data Saved for Each Run

Field Type Purpose
run_id UUID Unique identifier for the run (used in WebSocket URL)
agent_name string Which agent executed (strategist, creative, analyst, etc.)
input_params JSON Complete input parameters for audit trail
success boolean Whether the run completed without errors
output JSON Complete agent output (strategy, creatives, recommendations, etc.)
error string Error message if success=false
tool_calls_made JSON array All tools executed with inputs/outputs (observability)
total_input_tokens integer Total tokens sent to Claude API
total_output_tokens integer Total tokens generated by Claude
latency_ms float End-to-end execution time in milliseconds
trace_url string Link to Langfuse trace (if enabled)

Common Issues & Troubleshooting

Issue: "Failed to connect to WebSocket"

Cause: API server not running on correct port

Fix: Verify backend is running: curl http://localhost:8001/health

Issue: "API error: 500"

Cause: Agent initialization or execution error

Fix: Check backend logs: tail -50 /tmp/backend.log

Issue: "PostgreSQL connection failed" in logs

Cause: Database not running (expected in dev environment)

Fix: This is non-blocking. Runs still execute but aren't persisted to DB.

Issue: "ChromaDB connection failed" in logs

Cause: Vector database not running

Fix: Agents won't have access to knowledge base, but can still execute with basic reasoning.

Production Deployment Checklist

Infrastructure

  • Deploy frontend to CDN (Vercel, Netlify, AWS CloudFront)
  • Deploy backend to container service (Docker on AWS ECS, Google Cloud Run, Render)
  • Set up PostgreSQL (AWS RDS, Google Cloud SQL, or self-managed)

  • Configure Langfuse (optional, for observability)

Security

  • Enable HTTPS for all endpoints
  • Set up API authentication (JWT, OAuth2)
  • Configure CORS properly (restrict to your domain)
  • Secure database credentials (use environment variables, AWS Secrets Manager)
  • Enable rate limiting on API endpoints
  • Set up WAF (Web Application Firewall)

Monitoring & Observability

  • Set up centralized logging (ELK, CloudWatch, Datadog)
  • Configure APM (Application Performance Monitoring)
  • Set up alerting for critical errors
  • Monitor API latency and throughput
  • Track token usage for cost management
  • Monitor database performance

Scaling

  • Enable horizontal scaling for backend API
  • Set up load balancing
  • Configure auto-scaling policies
  • Implement connection pooling for database
  • Cache frequently accessed data (knowledge base, benchmarks)
